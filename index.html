Trema?

Tremaとは

Trema is an OpenFlow controller programming framework that provides everything needed to create OpenFlow controllers in Ruby.

GitHub - trema/trema

Tremaは、RubyでOpenFlow/SDNコントローラーを作成するためのフレームワーク

OpenFlow/SDN?

SDNとは

SDNはネットワーク制御機能とデータ転送機能が分離し、プログラムによりネットワークの制御が実現できる、新しいアプローチのネットワークです。

ONF - OPEN NETWORKING FOUNDATION

OpenFlowとは

OpenFlow is the first standard communications interface defined between the control and forwarding layers of an SDN architecture.

ONF - OPEN NETWORKING FOUNDATION

OpenFlowは制御モジュールと転送モジュール間の通信インタフェースの定義で、SDNの最初の規格です。

Legacy

  • モジュール
    「転送モジュール」「制御モジュール」ともネットワーク機器上で動作
  • 経路
    ネットワーク機器がネットワークプロトコルで経路情報を収集し、経路計算する

OpenFlow/SDN

  • モジュール
    「制御モジュール」をネットワーク機器から分離し、コントローラーで動作
  • 経路
    OpenFlow/SDNコントローラーがネットワークトポロジーを把握し、経路計算する

宛先 MAC アドレス「01:80:C2:00:00:00」のパケットを落とすフローエントリをスイッチに登録。

learning switch

  def packet_in(_datapath_id, packet_in)
    @fdb.learn packet_in.source_mac, packet_in.in_port
    flow_mod_and_packet_out packet_in
  end

  def flow_mod_and_packet_out(packet_in)
    port_no = @fdb.lookup(packet_in.destination_mac)
    flow_mod(packet_in, port_no) if port_no
    packet_out(packet_in, port_no || :flood)
  end

パケットの送信元MACアドレスとパケットを受信したポート番号をFDB に登録(@fdb.learn)
パケットの宛先MACアドレスからパケットの送信をポート番号を検索(@fdb.lookup)
パケットの転送フローエントリーをスイッチに登録。
packet_inでコントローラーに転送されてきたパケットを転送

learning switch

  def flow_mod(packet_in, port_no)
    send_flow_mod_add(
      packet_in.datapath_id,
      match: ExactMatch.new(packet_in),
      actions: SendOutPort.new(port_no)
    )
  end

  def packet_out(packet_in, port_no)
    send_packet_out(
      packet_in.datapath_id,
      packet_in: packet_in,
      actions: SendOutPort.new(port_no)
    )
  end
